# FAC Platform V5.1 - 战略重启版更新日志

## 版本标识
**V5.1-ALPHA** - 与 2.0 僵尸版本的明确区分

## 一、核心架构重构

### 1.1 身份系统重构 (P0 优先级)
- **废除** 2.0 时代的「雇主/专家」二元论
- **引入** Party A (甲方/需求方) / Party B (乙方/提供方) 模式
- **身份流动性**：同一账号可在甲乙双方间自由切换
- **核心动作**：
  - 甲方：发布意图 → 锁定 30% 订金 → 验收交付
  - 乙方：维护能力矩阵 → 接收邀约 → 授权隐私 → 执行获酬

### 1.2 P0 导航修复
- **Logo 点击行为**：
  1. 重置意图
  2. 重置身份
  3. 回归首页

## 二、原子化能力矩阵 (Skill Matrix)

### 2.1 去行业化
- 打破「金融」「法律」等行业壁垒
- 将经验拆解为具体「专业特长」：
  - 遗嘱规划
  - ISO 9001 审核
  - 粤语同声传译
  - 跨境贸易合规
  - etc.

### 2.2 能力标签系统
```typescript
interface SkillTag {
  id: string;

  weight: number;          // AI 权重 (0-100)
  category: SkillCategory; // 能力类别

```

## 三、Smart Escrow 交易机制

### 3.1 30% 订金托管
- 甲方确认合作时，必须锁定 30% 订金至平台
- 乙方看到订金锁定后才启动工作
- 订金具备「保障金」性质，防范甲方无故毁约

### 3.2 非雇佣契约框架
- **全面清除**「招聘、薪资、职位」等字眼
- 所有交易定义为「自雇人士服务采购」
- 甲方不承担雇主法律义务

### 3.3 零佣金政策
- 平台不从乙方劳动报酬中抽成
- 仅收取固定低额行政费 (HKD 50/笔)
- 费用透明，用于支付云端成本

## 四、Privacy Guard 隐私授权机制

### 4.1 预脱敏原则
- 联系信息（电话、Email、WeChat）默认隐藏
- API 返回的数据必须已脱敏
- **严禁**在 HTML 源码中留存真实资料

### 4.2 脱敏示例
```
真实: 65432107 → 脱敏: 6543****
真实: alice@example.com → 脱敏: a***@example.com
真实: wxid_alice123 → 脱敏: wxid_****
```

### 4.3 授权机制
- 仅当乙方**主动点击「授权揭露」**后，甲方可看见真实联系方式
- 授权有效期：30 天
- 乙方可随时撤销授权

## 五、数据持久化 (Cloudflare D1)

### 5.1 数据库架构
- **废除** localStorage 存储核心数据
- **迁移**至 Cloudflare D1 (SQLite)
- **关键记录** (30% 订金等) 永不丢失

### 5.2 核心数据表
- `users` - 用户信息
- `skill_tags` - 能力矩阵
- `privacy_authorizations` - 隐私授权
- `tasks` - 任务/委托 (含 Smart Escrow)
- `transactions` - $FAC 代币流水
- `contact_info` - 联系信息 (预脱敏)

### 5.3 API 端点
```
GET    /api/users              # 用户列表 (分页)
GET    /api/users/:id          # 用户详情
PUT    /api/users/:id/role     # 切换身份
GET    /api/users/:id/skills   # 能力矩阵
POST   /api/users/:id/skills   # 添加能力

GET    /api/tasks              # 任务列表 (分页)
POST   /api/tasks              # 创建任务
PUT    /api/tasks/:id/lock-deposit  # 锁定 30% 订金
PUT    /api/tasks/:id/accept   # 乙方接受
PUT    /api/tasks/:id/deliver  # 提交交付
PUT    /api/tasks/:id/complete # 验收完成

GET    /api/privacy/contact/:userId    # 获取脱敏联系信息
POST   /api/privacy/authorize          # 授权隐私
GET    /api/privacy/reveal/:userId     # 获取真实联系信息 (需授权)
```
