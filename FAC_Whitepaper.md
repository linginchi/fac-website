# FAC (Facilitating Artisan Collective) 最終開發總綱

**版本：V1.9**

---

## 壹、核心交互：語音驅動的「萬能指揮框」

- **定位**：全自動意圖識別與語音導航入口。
- **交互原則**：「動口不手動」——能說話就不用打字，能確認就不用填表。
- **品牌延續**：智慧沈澱與傳承的 **Web3 去中心化對接平台**；隱私隱喻為「銀行私人保險箱 / 個人智慧錢包」。

---

## 貳、智能邏輯與功能路徑 (The Intelligent Flow)

### 1. 意圖自動識別 (Intent Classification)

- **技術實現**：接入多模型 Agent（如 Sonnet 4.6 + GPT-4o）實時分析。
- **需求方 (Demand)**：輸入包含「我想搵人」、「需要顧問」、「項目外判」等 → 自動切換至 **雇主模式**。
- **供應方 (Supply)**：輸入包含「我是專家」、「想接 Job」、「過兩招」等 → 自動切換至 **專家模式**。

### 2. 語音身份識別與註冊 (Voice-First Identity)

- **身份識別**：支持語音輸入用戶名/密碼，或通過語音特徵（Voice ID）識別回訪用戶。
- **新用戶引導**：若識別為新用戶，Agent 會溫柔提示：*「看來您是新朋友，我是您的 AI 秘書，請告訴我您的稱呼或直接授權 LinkedIn 註冊。」*
- **數據導出與確認**：
  - 用戶口述完需求或資歷後，Agent 自動生成一份**結構化摘要（Draft）**。
  - **語音確認機制**：Agent 詢問 *「以上資訊是否正確？」* 用戶回答「係、正確、沒問題」即視為完成提交。

---

## 參、技術指標：專業級語音處理

- **語音識別 (STT)**：集成 OpenAI Whisper 或同級別專業引擎，確保能聽懂**廣東話、英文及普通話**的混合表述（Code-switching）。前端可選用瀏覽器 Web Speech API 作為即時預覽。
- **反饋美學**：語音輸入時，萬能框呈現**動態波形圖（Waveform）**；確認成功時有**清脆的音效回饋**。
