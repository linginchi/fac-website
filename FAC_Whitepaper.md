# FAC (Facilitating Artisan Collective) 最終開發總綱

**版本：V1.8**

---

## 壹、品牌核心與定位

- **定位**：智慧沈澱與傳承的 **Web3 去中心化對接平台**。
- **隱私隱喻**：「銀行私人保險箱 / 個人智慧錢包」級別的隱私保護與價值密度。
- **口號**：*「有些答案，不在數據庫，在老江湖的腦袋中。」*

---

## 貳、身份與註冊：LinkedIn 優先策略

- **首選入口**：鼓勵用戶使用 **LinkedIn 帳號一鍵註冊**。
- **智慧導入 (One-Click Sync)**：用戶授權後，系統自動將 LinkedIn 上的**經歷、技能、職稱**複製到用戶的 **「個人智慧錢包（Private Vault）」** 中。
- **驗證機制**：透過 LinkedIn 數據為專家進行初步的「職稱真實性」背書，增加平台互信。

---

## 參、UI/UX 核心：萬能指揮框 (The Command Box)

- **視覺規範**：模仿 Google 極簡搜索框，置於首屏中心。
- **功能**：文字、語音、多模態上傳；Agent 驅動的交付目標導向（將模糊輸入轉化為標準化 Job Offer）。

---

## 肆、$FAC Token 經濟學（更新版）

### 1. 獎勵規則 (Reward Logic)

| 行為項目 | 獎勵金額 ($FAC) | 說明 |
|----------|-----------------|------|
| **LinkedIn 註冊獎勵** | **80** | 高於一般註冊，獎勵使用公信力帳號接入 |
| **LinkedIn 數據同步** | **50** | 授權將經歷導入個人錢包，完成資料完善 |
| **完善個人保險箱** | **20 － 100** | 每增加一個專業標籤 (+20)；上傳加密實戰案例 (+100) |
| **訂閱繳費返還** | **50 － 500** | 根據等級（白銀/黃金/鑽石）按月返還 |
| **智慧貢獻** | **50 － 200** | 提供建議、市場情報獲採納 |

### 2. 消耗與解碼 (Utility Logic)

| 項目 | 消耗 ($FAC) |
|------|-------------|
| 基礎行情解碼 | **10** / 次 |
| 深度資訊解碼 | **50** / 次 |
| 開啟私密對話 | **100** / 次 |

*防禦機制（鎖定期、品質門檻、回購與銷毀）見 [docs/token-rules.md](docs/token-rules.md)。*

## 伍、管理後台 (Admin Command Center)

- **內容管理 (CMS)**：更新文案與專業領域標籤。
- **統計看板**：訂閱增長曲線、**LinkedIn 註冊佔比監控**、$FAC Token 流轉。
- **隱私審核**：管理「錢包」數據的公開狀態。
- **獎勵發放控制台**：管理員一鍵審核並撥發獎勵。
